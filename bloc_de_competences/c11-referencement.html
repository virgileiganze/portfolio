C11

Référencement des services en ligne et mesure de leur visibilité

Réalisation en milieu professionnel — Hackathon BTS SIO

Dans le cadre du hackathon, j'ai participé à la mise en ligne et au suivi de la visibilité du site vitrine de l'entreprise fictive Altis, développé en HTML, CSS et JavaScript.

Publication via GitHub Pages : le site a été rendu accessible publiquement en suivant les étapes suivantes :

  • Création d'un repository public sur GitHub
  • Push des fichiers du site (index.html, styles.css, scripts)
  • Activation de GitHub Pages depuis les paramètres du repository (branche main)
  • Obtention d'une URL publique accessible depuis n'importe quel navigateur
Mécanisme GitHub Pages : à chaque push sur la branche configurée, GitHub déclenche un pipeline de build qui déploie automatiquement les fichiers statiques sur ses serveurs CDN. L'URL publique est résolue via DNS vers les serveurs GitHub, qui servent le contenu en HTTPS.

Mesure de la visibilité avec Google Analytics : pour suivre la fréquentation du site, Google Analytics (GA4) a été intégré en ajoutant le tag de suivi (gtag.js) dans le <head> de chaque page. Les indicateurs suivis incluaient :

  • Nombre de visiteurs uniques et sessions
  • Pages les plus consultées
  • Sources de trafic (direct, réseaux sociaux, recherche)
  • Durée moyenne des sessions
Limite : la démarche couvre la publication et la mesure d'audience. Le référencement naturel (SEO) — balises meta structurées, sitemap.xml, indexation via Google Search Console — n'a pas été mis en œuvre dans le cadre de ce projet.
