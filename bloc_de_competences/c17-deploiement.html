Déploiement de GLPI — Installation from scratch

Réalisation en milieu professionnel — Hackathon BTS SIO

J'ai installé et configuré GLPI de manière complète sur un serveur dédié au projet, en partant d'un environnement vierge. L'objectif était de disposer d'un outil de gestion du parc et des incidents opérationnel, accessible à l'ensemble de l'équipe pendant le hackathon.

Stack déployée :

  • Serveur web Apache — configuration du virtual host pour servir l'application GLPI
  • PHP — installation des extensions requises par GLPI (php-mysql, php-xml, php-mbstring, etc.)
  • MySQL — création de la base de données et de l'utilisateur GLPI avec les droits appropriés
  • Agent GLPI — déploiement sur les postes pour la remontée automatique d'inventaire
Pourquoi cette stack ? GLPI est une application PHP qui nécessite un serveur web (Apache ou Nginx), un interpréteur PHP et une base de données relationnelle (MySQL/MariaDB). L'installation from scratch implique de configurer chaque composant manuellement, contrairement à une installation via un package préconfiguré.

Une fois le serveur opérationnel, GLPI a été rendu accessible aux membres de l'équipe via le réseau local, permettant la création et la gestion de tickets d'incidents en conditions réelles pendant le hackathon.

Déploiement du site vitrine Altis — Mise en production sur Alwaysdata

Réalisation en milieu professionnel — Hackathon BTS SIO

Le site vitrine de l'entreprise fictive Altis, développé en HTML, CSS et JavaScript, a été mis en production sur la plateforme d'hébergement Alwaysdata afin de le rendre accessible via internet.

Processus de déploiement :

  • Connexion au serveur via FileZilla (client FTP)
  • Transfert des fichiers du site vers le répertoire web du serveur Alwaysdata
  • Vérification du bon fonctionnement du site en conditions réelles (accès URL, rendu navigateur)
FTP vs SFTP : le transfert via FTP (FileZilla) est fonctionnel mais non chiffré — les identifiants et fichiers transitent en clair sur le réseau. En environnement de production réel, le protocole SFTP (FTP over SSH) est préférable car il chiffre la connexion. Alwaysdata supporte SFTP nativement.

Le site développé comprend une page d'accueil avec présentation de l'équipe, une section avis avec gestion dynamique côté client (JavaScript), et une intégration Google Analytics pour le suivi de fréquentation.

⚠ Le formulaire d'avis est géré entièrement côté client (JavaScript) : les données ne sont pas persistées en base de données. Les avis s'affichent dynamiquement dans le DOM mais disparaissent au rechargement de la page. Une évolution possible serait d'ajouter un backend PHP avec insertion MySQL.
